## Epic 3. B2B 파트너 콘솔 (관리자 화면)

**운영 방침:** 단순한 광고 성과 측정을 넘어, 점주가 비즈니스 인사이트를 얻을 수 있는 '고객 취향 데이터'를 제공하여 플랫폼 의존도를 높인다.

> ⚠️ **미확정 (P-5):** 초기 도입기 풀 스펙 무료 제공 → 안정기 프리미엄 플랜 분리 전환 시점 및 기준

---

### 1. 사먹기 파트너 (식당 점주) 대시보드

**1.1 핵심 성과 지표 (KPI)**

| 지표 | 내용 |
|---|---|
| 노출 수 (Impressions) & 클릭 수 (Clicks) | 일반 마커 및 챌린지 광고(별표 마커) 핀의 노출 및 클릭 성과 |
| 실제 주문 전환 수 및 전환율 | 길 찾기 클릭, 예약 연동 등 실질적인 방문/주문으로 이어진 비율 |
| 평균 체류 시간 (Dwell Time) | 식당 상세 페이지 진입 후 이탈하기까지의 시간 (바운스 레이트 추적용) |

**1.2 오디언스 취향 인사이트 (Audience Preference Analytics)**

- **메뉴별 타겟 분석:** 특정 메뉴(예: 평양냉면)를 클릭한 유저들의 평소 입맛 성향 데이터(예: '슴슴한 맛 선호', '육향 중시')를 익명화하여 리포트로 제공

> ⚠️ **미확정 (B-5):** 오디언스 취향 인사이트 데이터 익명화 처리 방식 (개인정보보호법 검토 필요)

---

### 2. 해먹기 파트너 (소형 마트 점주) 주문 처리 시스템

**운영 방침:** 신규 장비 도입 허들을 없애고, 한국 표준 배달 앱(예: 배달의민족 등) 수발주 프로세스와 유사한 UX 제공 → 점주 학습 비용 최소화.

**2.1 주문 인입 및 알림 (Order Notification)**

| 항목 | 정책 |
|---|---|
| 청각/시각 알림 | 주문 인입 시 명확한 전용 사운드(알림음) + 화면 팝업 제공 |
| 하드웨어 호환성 (Phase 1) | 기존 점포의 PC 웹브라우저, 태블릿, 스마트폰 앱 환경에서 완벽 구동. 크로스 플랫폼 지원 |

> ⚠️ **미확정 (B-1):** [Tech Spike/추후 과제] 기존 POS 시스템 및 영수증 프린터(Serial/Bluetooth)와의 직접 연동 모듈 개발 여부 — Phase 1 이후 검토

**2.2 주문 상태 관리 및 고객 안내 (Order Fulfillment Flow)**

**배달 주문 (Delivery)**
```
점주가 [배달 출발] 버튼 클릭
    ↓
예상 소요 시간(ETA) 직접 입력 팝업 노출
    ↓
고객에게 배달 시작 알림톡 발송
```

**픽업 주문 (Pick-up)**
```
점주가 [픽업 준비 완료] 버튼 클릭
    ↓
고객에게 자동 안내 메시지 발송
("도착 시 주문번호 [0000]번을 말씀해 주세요")
```

## Epic 4. 프리미엄 그룹 추천 및 스페이스 초대 플로우 (BM 연계형)

**기능 정의:** 2인 이상 모임 시, 솔로 탐색과 분리된 진입점(상단 프로필 ⊕ 버튼)을 통해 그룹원의 식사 이력과 취향을 교차 분석하여 '최적의 교집합 식당'을 추천하는 기능. **(서비스 핵심 BM)**

---

### 1. 그룹 초대 및 Web-to-App 온보딩 퍼널

**트리플 초대 시스템**

| 방식 | 내용 |
|---|---|
| 카카오톡 1-Click 초대 링크 | 가장 빠른 초대 방식 |
| 스페이스 입장 코드 | 4~6자리 코드 입력 |
| 앱 내 ID 검색 | 직접 검색 후 초대 |

> ⚠️ **미확정 (B-6):** 스페이스 입장 코드 자릿수 ("4~6자리" — 확정 전)

**웹 뷰(Web View) 진입 허용**
- 앱이 없는 유저가 초대 링크 클릭 시 → 모바일 웹 페이지로 랜딩 → 즉시 그룹 합류 및 입맛 테스트 가능

**점진적 전환 유도 (Soft Gating)**
- 웹 환경에서는 필수 추천 과정만 진행
- 결과 확인 전후로 "기록을 남기고 다음엔 더 찰떡같은 추천을 받으려면 로그인(앱 설치)하세요!" 안내
- B2B 마트 연계 등은 앱 전용 기능임을 명시하여 설치 유도

---

### 2. 그룹 추천 알고리즘 필터링 우선순위

| 순위 | 유형 | 내용 |
|---|---|---|
| 1순위 (Hard Filter) | 절대 제외 | 단 1명이라도 가진 '알러지 유발 식재료'나 '약물 복용 금기 식재료'가 포함된 메뉴/식당 무조건 제외 |
| 2순위 (Deduplication) | 중복 방지 | 그룹 멤버들이 최근 24시간 내에 먹었던 메뉴나 식당 후순위 배제 |
| 3순위 (Soft Intersection) | 무난한 교집합 | 특정인의 극단적인 입맛에 치우치지 않고, 구성원 모두가 '보통' 이상의 긍정도를 보일 수 있는 교집합 탐색 |

> ⚠️ **미확정 (B-4):** 그룹 중복 방지 기준 시간 ("최근 24시간" — 확정 전)

---

### 3. 제안 방식

- **식당 단위 제안 (Restaurant-Centric Proposal):** 단일 메뉴명 통일이 아닌 [식당 이름] + [음식 종류 요약] 형태로 제공
- 크림 파스타 기피자도 같은 식당에서 토마토/오일 파스타를 고를 수 있도록 선택의 유연성 보장

---

### 4. 수익화(BM) 정책

- 어뷰징 방지 및 유료 플랜 전환을 위해 월간/주간 무료 사용 횟수와 최대 초대 가능 인원에 시스템적 제한을 둔다

> ⚠️ **미확정 (B-2):** 그룹 추천 월간/주간 무료 사용 횟수 제한 수치
> ⚠️ **미확정 (B-3):** 그룹 추천 최대 초대 가능 인원 수치
