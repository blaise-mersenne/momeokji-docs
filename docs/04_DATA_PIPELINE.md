## 1. 기초 데이터 파이프라인 (Base Data Automation)

서비스 초기 런칭 시점의 빈 깡통(Empty) 상태를 방지하기 위해 외부 데이터를 적극 연동한다.

### 1.1 Open API 연동

- 포털 지도 API(네이버/카카오/구글 등) 및 공공데이터 포털의 외식업 데이터 크롤링/연동
- 수집 대상: '식당 상호명, 기본 메뉴명, 가격' 등 변하지 않는 기초 뼈대 데이터를 1차로 세팅

> ⚠️ **미확정 (D-1):** 포털 지도 API 크롤링/연동 — 각 플랫폼별 API 이용약관 및 계약 조건 확인 필요. 사용 가능 범위 확정 전.

## 2. UGC(유저 생성 콘텐츠) 기반 마이크로 태그 추출 (Data Flywheel)

B2B 점주의 개입 없이, 앱을 사용하는 유저들의 맛 평가 활동 자체가 곧바로 식당의 '태그 자산'이 되도록 선순환 구조(Flywheel)를 만든다.

### 2.1 자연어 기반 태그 마이닝

- [Epic 2]의 상세 평가 단계에서 유저가 남긴 코멘트를 AI가 분석
- 예시: "이 집 김치찌개는 계란후라이를 줘서 너무 좋아요" → 해당 메뉴 DB에 `#계란후라이_제공` 마이크로 태그 자동 생성 및 매핑

> ⚠️ **미확정 (D-2):** UGC 자연어 태그 마이닝에 사용할 AI 모델 및 구체적 구현 방식

### 2.2 집단 지성 가중치 (Collective Intelligence Weighting)

- 특정 태그(예: 계란후라이)에 대한 다수 유저의 교차 검증(언급량 증가)이 누적될수록 해당 태그의 신뢰도 점수 상승
- 신뢰도 점수가 높은 태그 → [Epic 8] 동적 정렬 추천 시 강력한 부스터로 작용

> ⚠️ **미확정 (D-3):** 태그 신뢰도 점수 집계 기준 수치 (언급 횟수 임계값 등 — 확정 전)

## 3. B2B 파트너(점주) 자율 개입 및 보상 정책 (Opt-in B2B Console)

점주의 데이터 입력은 필수가 아닌 **'특권(선택)'** 으로 포지셔닝한다.

### 3.1 히든/특수 메뉴 어필

- 공공 API나 지도에 노출되지 않는 점주만의 비밀 메뉴, 신메뉴 등을 직접 태그로 등록할 수 있는 창구 제공 (원하는 점주에 한해)

### 3.2 행동 유도 보상 (Incentive / Gamification)

- 점주가 직접 디테일한 태그(식재료, 맵기 등)를 보강할 경우 → 유저 검색 시 매칭 알고리즘 정확도 향상 → '우리 식당의 상위 노출 확률(무료 광고 효과)' 대폭 상승
- 이 효과를 **게이지 애니메이션** 등으로 직관적으로 시각화하여 자발적 참여 유도

> ⚠️ **미확정 (D-4):** 게이지 애니메이션 보상 UI 구체적 구현 방식 및 디자인

## 4. B2C → B2B 데이터 플라이휠 전체 흐름

```
[수집 — B2C]
유저가 메인 홈 화면에서 식당 카드의 취향 태그(#계란후라이, #매콤달콤 등)를
열람(아코디언 +2 뱃지 오픈) 시 조회/관심 트래픽을 DB에 기록
        ↓
[가공 — Server]
특정 상권 내 유저들의 취향 태그 선호도 집계
개별 식당이 보유한 태그가 상권 전체에서 차지하는 매력도(%) 연산
        ↓
[출력 — B2B]
GET /api/v1/b2b/insights 호출
→ 퍼센티지 데이터를 B2B 대시보드 위젯(가로 막대그래프)에 렌더링
```

### API 명세 (B2B 인사이트 엔드포인트)

| 항목 | 내용 |
|---|---|
| Endpoint | `GET /api/v1/b2b/insights` |
| 호출 시점 | B2B 사장님 대시보드 진입 시 |
| 응답 데이터 | 상권 대비 태그별 매력도 퍼센티지 |
| 렌더링 UI | 가로 막대그래프 위젯 |
