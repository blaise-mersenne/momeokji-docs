## Epic 5. 메인 홈 및 내비게이션 UX (Zero-Friction 탐색)

**기능 정의:** 앱 진입 시 복잡한 조건 설정 없이, 중앙의 버튼 하나로 즉각적인 개인 맞춤 메뉴를 제안하는 몰입형 코어 UX 및 유저 상태별 맞춤 탐색 로직.

### 1. 글로벌 내비게이션 정책 (GNB-less UI)

- **코어 화면:** 앱 실행 시 즉각적으로 '추천 홈' 단일 화면만 노출하여 시선 집중
- **부가 메뉴 수납:** '마이페이지', '환경설정' 등 서브 메뉴는 우측 상단 햄버거 버튼(☰) 또는 프로필 아이콘 내에 수납(Hidden Navigation) → 시각적 노이즈 제거

### 2. 홈 진입 시 큐레이션 로직 (Cold/Warm Start Policy)

| 유저 상태 | 처리 방식 |
|---|---|
| Cold Start (신규/데이터 부족) | 아키네이터(Akinator) 방식의 동적 질의응답 제공 (예: "오늘 스트레스받는 일이 있었나요?") → 최적 리스트 도출 |
| Warm Start (데이터 누적) | 중앙의 [뭐먹지?] 버튼 클릭 한 번으로 과거 데이터와 맥락 분석 → 즉시 리스트 렌더링 (Zero-Friction) |

- **점진적 정보 노출:** 버튼 하단 [특별히 원하시는 게 있나요?] 텍스트 → 클릭 시에만 입력창 활성화 → 자연어로 추가 조건 입력

### 3. 양방향 탐색 필터링 및 Dimmed UX 정책

- **상태 전환:** 홈 화면 상단 또는 플로팅 영역에 [사먹기], [해먹기] 토글 버튼 배치

**조건부 Dimmed(음영) 처리 로직:**

| 모드 | Dimmed 기준 |
|---|---|
| 사먹기 | 도보/차량 이동에 물리적으로 너무 먼 거리의 식당/메뉴 → Dimmed |
| 해먹기 | 가상 냉장고 데이터 기준, 보유하지 않은 식재료 비중이 50%를 초과하는 레시피 → Dimmed |

> ⚠️ **미확정 (U-1):** 해먹기 Dimmed 처리 기준 수치 ("50% 초과" — 확정 전)

- **유저 선택권 보장 (Override Action):** Dimmed 처리된 항목도 탭(클릭) 허용. 유저의 강력한 의지를 차단하지 않음.

## Epic 6. 회원가입 및 온보딩/프로파일링

**기능 정의:** 가입 장벽을 낮추고 유저의 입맛 취향을 지속적으로 수집하는 시스템.

### 1. 계정 및 로그인 정책 (Authentication & Guest Mode)

**1.1 소셜 로그인 전용 (OAuth-Only)**
- 카카오, 네이버, 구글, 애플 연동을 통한 1초 간편 가입만 지원
- 자체 이메일 가입 폼 미제공

**1.2 지연된 회원가입 (Deferred Registration) 및 비회원 플로우**

| 항목 | 정책 |
|---|---|
| 비회원 진입 | 앱 최초 실행 시 로그인 강제하지 않음. Guest Mode에서도 '메뉴 추천' 1회 사이클 완벽 경험 가능 |
| Soft Gating 시점 | 1회 추천 경험 후 다음번 앱 진입 시, 또는 두 번째 추천 시도 시 Bottom Sheet 노출 |
| 카피 정책 | "나에게 딱 맞는 메뉴 추천을 계속 받으시려면 회원 가입을 권장드려요!" (혜택 강조) |

> ⚠️ **미확정 (U-2):** 비회원 Soft Gating 정확한 발동 횟수 기준 (현재 "1회" — 확정 전)

### 2. 초기 프로파일링 및 상시 데이터 수집 UX (Continuous Profiling)

**2.1 하이브리드 입력 폼 (Hybrid Input)**

초기 프로파일링 화면 및 마이페이지 설정에서 두 가지 형태 입력 모두 수용:

| 입력 유형 | 예시 |
|---|---|
| 단답형/키워드형 | "견과류 알러지", "오이 제외" — 시스템 필터링에 직결되는 명확한 조건 |
| 자연어/문장형 | "신라면 정도의 매운맛이 좋아", "오늘은 국물이 끌려" — 맥락과 강도가 포함된 서술형 조건 |

**2.2 상시 소통형 플로팅 UI (Always-on Profiling FAB)**

- **위치 및 디자인:** 홈 화면 우측 하단에 챗봇이나 말풍선 형태의 FAB 상시 고정 배치 ("말을 걸어보자" 뉘앙스)
- **동작 정책:** 유저가 언제든 탭하여 기분, 특수 상황(다이어트, 감기 등), 세밀한 입맛 취향 추가 입력 가능. 입력 데이터는 즉시 다음 추천 알고리즘 변수로 반영

> ⚠️ **미확정 (U-4):** FAB 버튼 최종 디자인 형태 (챗봇/말풍선 등 — 디자인 확정 전)

## Epic 7. 서비스 페르소나 및 예외 상황 방어 UX

**기능 정의:** 서비스 공식 화자의 톤앤매너를 규정하고, 검색 결과가 없거나 시스템 한계에 부딪히는 에러 상황(Edge Case)에서 유저 이탈을 방어하는 대화형 UX.

### 1. 서비스 공식 페르소나 및 톤앤매너

**1.1 코어 페르소나**
> '맛집 추천에 진심인, 예의 바르면서도 은근히 유쾌한 직장 동료/후배'
- 깍듯한 존댓말을 베이스로 하되, 가끔 툭툭 던지는 너스레로 유저(선배)에게 피식하는 웃음 유발

**1.2 UI 시각화 (Visual Representation)**
- 직장 생활의 에피소드를 감칠맛 나게 풀어내는 친근한 애니메이션 캐릭터
- 화면 한편 또는 팝업창에 가상의 2D 캐릭터(동료 아바타) 배치 → 텍스트 타격감과 몰입감 극대화

> ⚠️ **미확정 (U-3):** 2D 캐릭터(동료 아바타) 구체적 디자인 및 디자인 리소스 확보 여부

**1.3 과금(BM) 유도 톤앤매너**
- 기능 제한에 대한 딱딱한 통보가 아닌, 수고에 대한 귀여운 보상 요구 형태로 접근
- 예시: "저의 메뉴 추천, 만족스러우셨습니까..? 그렇다면 열일한 저에게 커피 한 잔.. 어떠신지 (찡긋)"

### 2. 핵심 예외 케이스(Edge Case) 및 Empty State 방어 시나리오

**2.1 추천 결과가 0건일 때 (과도한 필터링 예외)**

- **상황:** 알러지, 매운맛 기피, 거리 제한 등 조건이 너무 빡빡하여 교집합이 없을 때
- **방어 카피:** "선배님, 제 미식 레이더가 멈춰버렸습니다..! 조건을 딱 하나만 양보해 주시면, 제가 다시 땀나게 뛰어보겠습니다 🏃‍♂️💦"
- **Action 유도:** [거리 1km 넓히기], [조건 하나 빼기] 버튼 노출

**2.2 주변에 오픈한 식당이 전무할 때 (위치/시간 예외)**

- **상황:** 심야 시간이거나 외곽 지역이라 '사먹기' 반경 내 식당이 없을 때
- **방어 카피:** "이 근처는 다들 일찍 퇴근했나 봅니다. 🌙 대신 제가 당장 배달되는 곳을 뚫어보거나, 선배님 냉장고를 털어볼까요?"
- **Action 유도:** [배달로 전환], [해먹기로 전환] 버튼 노출

**2.3 도저히 타협이 안 되는 그룹 추천 (프리미엄 BM 예외)**

- **상황:** 일행 간의 알러지나 기피 식재료가 완벽하게 충돌하여 추천이 불가능할 때
- **방어 카피:** "선배님들.. 모두의 입맛 평화 협정에 실패했습니다 🏳 오늘은 각자 드시고 싶은 걸 시키시되, 제 추천 메뉴도 꼭 한 입씩 나눠 드셔보시는 건 어떨까요? (씨익)"
- **Action 유도:** [다건 배달/푸드코트 제안] 형태의 대안 화면 제공

**2.4 가상 냉장고가 텅 비었을 때 (해먹기 결품 예외)**

- **상황:** 입력/추적된 식재료가 없어 레시피 추천이 불가능할 때
- **방어 카피:** "선배님, 냉장고에서 요정이 나올 것 같습니다! 🧚 물만 드실 순 없으니, 당장 요리 가능한 밀키트를 제가 퀵으로 대령할까요?"
- **Action 유도:** [근처 마트 즉시 배달 장바구니 연동] 버튼 노출
